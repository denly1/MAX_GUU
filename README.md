# Бот | Обучение служением | ГУУ

Чат-бот для платформы **MAX** ([max.ru](https://max.ru)), автоматизирующий
коммуникацию участников программы «Обучение служением» в ГУУ: регистрацию,
верификацию, выбор социальных задач, сбор заявок и обратной связи, рассылки и
напоминания о созвонах.

Реализован на Python с использованием библиотеки
[`maxapi`](https://github.com/love-apples/maxapi) (стиль aiogram: Router, FSM,
inline-клавиатуры). Данные хранятся в SQLite.

## Возможности

- **До регистрации:** часто задаваемые вопросы, «задать свой вопрос»,
  справочная информация об администраторах, регистрация.
- **Регистрация** студентов, преподавателей и социальных заказчиков с
  валидацией ввода (раздел 4 ТЗ).
- **Верификация** пользователей администраторами (кнопки «Подтвердить /
  Отклонить»).
- **Выбор социальной задачи** студентами: создание команды (лидер) с паролем,
  вход в команду (участник) по паролю, выбор и подтверждение задачи,
  уведомление команды.
- **Управление задачами** администратором: добавление, активация/скрытие,
  удаление, выгрузка списка задач в Excel.
- **Выгрузка выбора проектов** в Excel (раздел 2.5).
- **Рассылка приглашений** на мероприятия с кнопками подтверждения участия.
- **Напоминания о созвонах** для команды / всех студентов / всех пользователей
  + просмотр подтверждений (`/responses <id>`).
- **Мемы по кодовому слову** + управление мемами администратором.
- **Обратная связь** и ответы администратора (`/answerq`, `/answerf`).
- **Заявки на реализацию проекта** (раздел 2.12) для преподавателей и
  социальных партнёров; просмотр заявок администратором.

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows (PowerShell: .venv\Scripts\Activate.ps1)
pip install -r requirements.txt
```

## Настройка

1. Скопируйте `.env.example` в `.env`.
2. Укажите `MAX_BOT_TOKEN` (токен бота из консоли разработчика MAX).
3. Запустите бота, напишите ему `/id`, получите свой `user_id`.
4. Впишите свой `user_id` в `ADMIN_IDS` (через запятую можно несколько).
   Эти пользователи становятся администраторами автоматически, без верификации.

```env
MAX_BOT_TOKEN=ваш_токен
ADMIN_IDS=123456789,987654321
```

## Запуск

```bash
python run.py
```

Бот работает в режиме long polling.

## Структура проекта

```
bot/
  config.py        — конфигурация (токен, ADMIN_IDS, пути)
  database.py      — схема SQLite и первичное наполнение (FAQ, контакты)
  repo.py          — функции доступа к данным
  texts.py         — статические тексты, ЧаВо, контакты, списки опций
  validators.py    — валидация ввода (ФИО, телефон, пароль, ссылка, ник)
  states.py        — FSM-состояния всех сценариев
  keyboards.py     — построители inline-клавиатур
  filters.py       — фильтр маршрутизации callback по префиксу
  common_ui.py     — главное меню, уведомления администраторам
  reports.py       — Excel-выгрузки (openpyxl)
  instance.py      — общий Bot и Dispatcher
  main.py          — точка входа
  handlers/        — обработчики по модулям (common, registration, faq,
                     verification, tasks, task_admin, mailings, memes,
                     feedback, applications)
run.py             — запуск: python run.py
```

## Примечания

- Картинки администраторов и шаблоны отчётности выдаются ссылками на
  Яндекс.Диск (как указано в ТЗ).
- Мем-картинка сохраняется по токену вложения MAX: администратор отправляет
  фото боту при создании мема.
- Лимит запросов к MAX — 30 rps; бот укладывается в него при типичной нагрузке.
